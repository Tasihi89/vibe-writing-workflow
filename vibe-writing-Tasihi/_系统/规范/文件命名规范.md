# 文件命名规范

为避免在Windows系统中出现文件名非法字符问题，特制定本命名规范。

## Windows系统禁止使用的字符

以下字符不能在文件名中使用：

- `<` (小于号)
- `>` (大于号)
- `:` (冒号)
- `"` (双引号)
- `/` (正斜杠)
- `\` (反斜杠)
- `|` (竖线)
- `?` (问号)
- `*` (星号)

## 输出卡片命名规则

输出卡片文件名采用以下格式：

```
[源知识卡片序号]-[卡片内部分割序号]-[主题内容不含非法字符].md
```

示例：

- `01-01-为什么Pod是Kubernetes的最小单位.md`
- `02-03-如何配置健康检查.md`

## 命名规则说明

1. **源知识卡片序号**：两位数字，与知识卡片文件名前的序号一致
2. **卡片内部分割序号**：两位数字，表示该知识卡片拆分成的第几个输出卡片
3. **主题内容**：去除所有非法字符，使用中文表述核心主题
4. **文件扩展名**：统一使用`.md`

## 注意事项

1. 文件名总长度不应超过255个字符
2. 避免使用特殊符号，推荐使用中文、英文字母和数字
3. 空格应尽量避免，如有需要可用连字符(-)替代
4. 文件名不得以点(.)开头
